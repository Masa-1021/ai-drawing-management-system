# Mitsubishi Electric Brand Design Skill

このスキルは、三菱電機のブランドイメージに沿った Web/UI デザイン・コード生成を行うためのガイドラインです。  
目的は「信頼性」「技術力」「精密」「上質」「堅牢さ」を感じる UI を安定して生成することです。

---

# 1. Brand Identity（ブランドアイデンティティ）

## 1.1 キーワード
- 信頼性
- 技術力・精密さ
- クリーン・整然
- 未来志向
- 上質・堅牢

## 1.2 トーン＆マナー
- 過度に派手にしない  
- 落ち着き + 力強さ  
- 規律・秩序を感じるレイアウト  
- 機械的・論理的で無駄がない構成

---

# 2. Color Palette（ブランドカラー）

## 2.1 コーポレートカラー（最重要）
- **ME Red（ブランド赤）**
  - HEX: `#FF0000`  
  - RGB: (255, 0, 0)  
  - 説明：三菱電機の象徴カラー。強調点、CTAボタン、アラート、重要なアクセントに使用。

## 2.2 グレー系（ベース）
- **Light Grey（背景）**: `#F2F2F2`
- **Medium Grey（線・UI境界）**: `#C4C4C4`
- **Dark Grey（本文テキスト）**: `#333333`
- **Deep Grey（見出し等）**: `#111111`

→ 三菱電機の「冷静・信頼・精密」イメージを作るベースカラー。

## 2.3 補助色（必要な場合のみ使用）
- **Blue（技術的・信頼）**: `#2A60AD`
- **Steel Blue（工業的ニュアンス）**: `#496781`
- **Green / Yellow / Orange**：アクセントとして弱く使用可

使いすぎは禁止。主役は赤とグレー。

---

# 3. Typography（フォント）

## 3.1 方針
- 工業的 × クリーン × 読みやすいフォント
- サンセリフを基本
- 過度に丸い・かわいいフォントは禁止

## 3.2 推奨フォント
- 日本語：  
  - **Noto Sans JP**  
  - **Source Han Sans (源ノ角ゴシック)**  
  - **Yu Gothic / Yu Gothic UI**

- 英文：
  - **IBM Plex Sans**
  - **Helvetica Neue**
  - **Inter（必要なら）**

---

# 4. Layout（レイアウト）

## 4.1 グリッド
- 8px グリッド / 4px サブグリッド
- 均等な余白
- 左右非対称や、カジュアルなレイアウトは禁止

## 4.2 コンポーネント形状
- 角丸は控えめ（2〜4px）
- ボタンはシャープ寄り
- 影は弱く、または無し

## 4.3 情報整理
- セクションごとに明確に区切る
- 重要情報は ME Red で示す
- 背景は白 or Light Grey を基本

---

# 5. UI Components（UI 部品のルール）

## 5.1 ボタン
- Primary: ME Red (#FF0000)
- Secondary: Dark Grey (#333333)
- Hover: 少しだけ明るくする（彩度を下げない）
- Padding は 12–16px で統一  
- Icon + Text の場合、間隔は 8px

## 5.2 カード
- 背景は白 or Light Grey
- 枠線は Medium Grey (#C4C4C4)
- 影は最小限

## 5.3 テーブル
- 工場・製造業向けっぽく「整然」を重視
- 行の背景は交互に `#FAFAFA`
- 行間は詰めすぎず読みやすく
- セル境界線は薄いグレー

---

# 6. Motion（アニメーション）

## 基本方針
- 過度なモーションは禁止
- 微細なトランジションのみ利用可

### 推奨：
- fade-in（100〜200ms）
- hover トランジション（150ms）
- リスト表示：ステガーアニメーション（軽く）

---

# 7. Background Design（背景）

## 7.1 原則
- 真っ白 or Light Grey が基本
- 強いグラデーション禁止
- 工業的な “静かなトーン差” は OK

## 7.2 使ってよい装飾
- スチール調の淡いパターン
- 幾何学的な薄いライン
- 工場の精密な雰囲気を壊さない程度の装飾

---

# 8. Copy & Tone（文言・表示テキストの方針）

- 丁寧、簡潔、明瞭
- 無駄な装飾語は使わない
- 堅牢で誠実な印象
- 半角英数の整った見出しも有効

---

# 9. Code（HTML/CSS/React）の原則

## 9.1 CSS の基本
- CSS 変数（:root）にブランドカラーを定義
- 例：

```css
:root {
  --me-red: #ff0000;
  --me-grey-light: #f2f2f2;
  --me-grey-medium: #c4c4c4;
  --me-grey-dark: #333333;
  --me-blue: #2a60ad;
}
````

## 9.2 Tailwind の場合

* カスタムテーマに ME Red を登録
* 角丸：`rounded-[2px]`
* 影：`shadow-sm` 以下のみ

## 9.3 React コンポーネント

* shadcn/ui を使う場合はブランド色を上書き
* 基本的に無駄なアニメーションは禁止

---

# 10. Claude に対する指示（Skill としての挙動）

* UI・デザインを生成する際は **必ずこのガイドの色・レイアウト・トーンを優先すること**
* AI 特有の “かわいい・丸い・カラフル” デザインは排除する
* ME Red を適切な比率で活用する
* 情報整理を明確にし、工業的・堅牢な雰囲気を出す
* コード生成では、ブランドカラーを CSS 変数化すること
* 背景は白または Light Grey を原則とする
